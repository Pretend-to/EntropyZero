
# EntropyZero UI/UX 设计文档 - 空间智慧任务引擎

## 0. 设计综述 (UX Vision Statement)

EntropyZero 致力于打破传统任务管理工具的线性束缚，提供一个**无界的工作空间 (Infinite Workspace)**，将任务、项目、知识以最直观的**空间关系**呈现。我们相信，通过将用户的心智模型映射到数字画布上，并辅以强大的 **AI 智慧伙伴**，用户能够以更自然、更少心智负担的方式组织和拓展复杂思维，将“任务列表”升级为“思维网络”。核心挑战在于如何将“无限”的自由与“有序”的结构巧妙结合，避免“无序黑洞”，而是构建一个“结构化自由”的智能环境。

## 1. 设计理念

### 1.1 核心原则 (Core Principles)

- **空间化思维 (Spatial Thinking)**: 利用二维空间表达所有信息（任务卡片、知识节点、空间区域）的关系，摆脱传统列表的线性限制，促进发散性思考和整体认知。
- **结构化自由 (Structured Freedom)**: 提供无限画布的自由度，同时通过“空间区域”和“AI 智能布局”提供按需的结构和秩序，避免用户迷失。
- **零延迟响应 (Zero Latency)**: 所有交互都应有毫秒级反馈，确保用户的思维流不被打断，无论内容多么复杂。
- **渐进式展示 (Progressive Disclosure)**: 复杂功能和大量信息通过聚焦、过滤和“透视视图”等方式渐进展现，降低认知负担。
- **AI 智慧辅助 (AI-Assisted Intelligence)**: 深度整合 AI 赋能智能布局、语义关联和内容生成，将繁琐的组织工作自动化，让用户专注于创造。
- **键盘优先 (Keyboard-First)**: 支持全键盘操作，提高效率用户的使用体验和操作速度。

### 1.2 视觉风格 (Visual Style)

- **极简主义 (Minimalism)**: 减少视觉噪音，突出内容本身和空间关系。
- **深色主题 (Dark Theme)**: 减少眼部疲劳，适合长时间专注使用，营造沉浸式体验。
- **微妙动效 (Subtle Motion)**: 使用物理感知的动画增强用户体验，尤其是在元素移动、视图切换时，提供“魔术感”和状态反馈。
- **高对比度 (High Contrast)**: 确保可访问性和清晰度，即使在复杂的画布上也能清晰识别元素。

## 2. 信息架构与核心概念 (Information Architecture & Core Concepts)

EntropyZero 的信息架构围绕一个统一的**工作空间 (Workspace)** 展开，所有的信息都存在于这个无限画布上，并通过以下核心概念进行组织和呈现。

### 2.1 核心概念澄清 (Refined Terminology)

为了解决“项目、视图、事项、无限画布”之间的混乱，我们明确以下核心术语：

1. **工作空间 (Workspace)**:

   * **定义**: 整个应用的主界面，是一个**单一的、无界限的、无限大的二维画布**。所有用户创建的信息（任务卡片、知识节点、空间区域）都驻留于此，拥有其全局坐标。
   * **UX 意义**: 统一的思维平台，避免信息孤岛和上下文切换成本。它是所有信息最真实的“家”。
2. **空间区域 (Space)**:

   * **定义**: 在“工作空间”中，用户可以自由地划定和命名的**逻辑容器**。一个“空间区域”可以是一个**项目、一个生活领域、一个主题、一个灵感板**等。它视觉上表现为画布上的可调整大小和位置的“框”。
   * **UX 意义**: 提供了在无限画布中**创建边界和结构**的方式，降低心智负担。用户可根据需求（项目、主题、时间段等）组织相关内容，并支持嵌套。AI 可识别其边界进行智能布局。
3. **任务卡片 (Task Card) / 知识节点 (Knowledge Node)**:

   * **定义**: “工作空间”中的基本信息单元。**任务卡片**代表待办事项、目标、计划等可执行项；**知识节点**代表笔记、文档链接、图片、Web Clipper 等信息资产。
   * **UX 意义**: 直观、原子化的信息表示，拥有丰富的属性（状态、标签、日期等），在画布上可自由移动和交互。
4. **关系连线 (Connection Line)**:

   * **定义**: 连接“任务卡片”、“知识节点”或“空间区域”之间的线条。用于可视化表达**依赖、因果、层级、相关性**等任意自定义关系。
   * **UX 意义**: 将散落的信息编织成网络，直观展示复杂项目的逻辑结构和思维流。
5. **透视视图 (Perspective View)**:

   * **定义**: 这是在“工作空间”中**动态生成、聚焦、筛选和重新布局**信息的方式。它不是一个独立的页面，而是**在当前画布上的一种智能呈现方式**。例如，AI 可以根据指令将特定“任务卡片”智能布局成看板、列表、甘特图、思维导图等风格。
   * **UX 意义**: 解决“无限画布”可能带来的无序感。用户可以随时请求 AI 或系统将特定条件下的信息以结构化的方式（例如“今日任务列表”、“本周项目 A 看板”）临时展示，而这些信息在数据层仍然保持其原始的画布坐标，并在退出视图时平滑返回。

### 2.2 核心界面结构 (Primary UI Structure)

```
EntropyZero Application
├── 工作空间 (Workspace) - 主画布区域
│   ├── 任务卡片 (Task Cards)
│   ├── 知识节点 (Knowledge Nodes)
│   ├── 空间区域 (Spaces) - 可嵌套
│   ├── 关系连线 (Connection Lines)
│   └── 浮动工具栏 (Floating Toolbar) - 包含缩放、定位、视图切换等快捷操作
├── 智能指令框 (Command Palette) - 全局操作入口
├── 侧边栏 (Sidebar) - 快速导航与管理
│   ├── 收藏的透视视图 (Saved Perspective Views) - 如“我的今日待办”、“项目X看板”
│   ├── 空间区域导航 (Space Navigation) - 树形/列表展示所有空间区域
│   ├── 快捷访问 (Quick Access) - 最近访问、标签管理、全局搜索历史
│   └── 设置面板 (Settings)
└── 状态栏 (Status Bar) - 同步状态、通知等

```

### 2.3 用户流程 (User Flows)

1. **首次使用**: 引导用户在工作空间中创建第一个**任务卡片**，并提示如何创建**空间区域**。
2. **日常任务管理**: 通过指令框快速创建和编辑**任务卡片**，在工作空间中拖拽组织，或激活“今日待办”**透视视图**聚焦。
3. **项目规划与深度工作**: 在**工作空间**中创建多个**空间区域**来代表不同项目，通过**关系连线**可视化任务依赖。使用 AI 辅助拆解任务和智能布局。
4. **回顾总结**: 放大到特定**空间区域**回顾项目进展，或激活自定义**透视视图**（如“本月已完成任务”）进行总结。
5. **知识管理**: 创建**知识节点**收集信息，并通过**关系连线**将其与**任务卡片**或其它**知识节点**关联，构建个人知识网络。

## 3. 界面组件设计

### 3.1 工作空间 (Workspace) - 无界画布

**功能**: 承载所有信息，实现空间化展示和交互的核心区域。

**设计要点**:

- 无限滚动的网格背景，支持自定义网格密度。
- 支持平滑的缩放 (10% - 500%)，以适应全局概览到细节聚焦。
- 平滑的拖拽平移和惯性滚动，提供自然的探索感。
- 视口外的元素可选择性淡化、隐藏或在小地图上显示。
- 小地图 (Mini-map): 可选择性显示在角落，提供整个工作空间的概览和快速定位。

**交互模式**:

- 鼠标中键/空格键 + 拖拽: 平移画布。
- 滚轮: 缩放。
- 双击空白区域: 创建新的**任务卡片**或**知识节点**（默认在当前视口中央）。
- 框选: 批量选择**任务卡片**、**知识节点**或**空间区域**。
- 多点触控: 捏合缩放、双指平移。

### 3.2 空间区域 (Space)

**功能**: 在工作空间中为信息提供逻辑边界和容器。

**视觉设计**:

```
┌───────────────────────────┐
│ 📁 项目A: 产品迭代计划         │ ← 标题（可编辑，包含图标）
│ ─────────────────────────── │
│ ┌───────────────────────┐ │
│ │ 🔵 前端任务1              │ │
│ └───────────────────────┘ │
│ ┌───────────────────────┐ │
│ │ 🟢 后端任务2              │ │
│ └───────────────────────┘ │
│                             │
└───────────────────────────┘
```

**设计要点**:

- 可拖拽调整大小的矩形区域，可自定义边框样式和背景色。
- 标题栏显示名称和图标，支持折叠/展开内部内容。
- 支持嵌套：一个**空间区域**内部可以包含其他的**空间区域**。
- 当拖拽**任务卡片**或**知识节点**进入/移出**空间区域**时，有清晰的视觉反馈。

**交互模式**:

- 拖拽边缘: 调整大小。
- 拖拽标题栏: 移动位置。
- 双击标题栏: 折叠/展开内容。
- 框选多个元素后，点击浮动工具栏“创建空间区域”按钮，或右键创建。
- 拖拽一个**空间区域**到另一个**空间区域**中，实现嵌套。

### 3.3 任务卡片 (Task Card) / 知识节点 (Knowledge Node)

**功能**: 单个信息单元的可视化表示。

**视觉层次**: (与原设计类似，但更强调可扩展性)

```
┌─────────────────────────┐
│ [●] 任务标题            │ ← 状态指示器 + 标题 (必填)
│ ─────────────────────── │
│ 任务描述内容...         │ ← 描述区域 (可折叠/按需显示)
│                         │
│ #标签1 #标签2          │ ← 标签区域
│ ⏰ 2024-01-15          │ ← 时间信息 (截止日期/创建日期等)
│ 🧑‍💻 张三                │ ← 负责人
│ ...更多自定义字段...    │
└─────────────────────────┘
```

**状态系统 (统一色彩语义)**:

- 🔵 待办 (Todo) - 蓝色
- 🟢 进行中 (In Progress) - 绿色
- 🟡 等待 (Waiting) - 黄色
- ✅ 完成 (Done) - 灰色
- 🔴 阻塞 (Blocked) - 红色

**知识节点特有功能**:

- 支持嵌入图片、视频链接预览。
- 支持富文本编辑。
- 支持网页内容剪藏 (Web Clipper)。

### 3.4 关系连线 (Connection Lines)

**功能**: 可视化表达信息单元间的依赖或关联关系。

**视觉设计**:

- 贝塞尔曲线，避免直角，确保流畅美观。
- 箭头指示方向（如依赖方向、信息流向）。
- 不同类型用不同样式或颜色:
  - **实线**: 强依赖 (前置任务必须完成)
  - **虚线**: 弱依赖 (建议顺序，非强制)
  - **点线**: 相关任务 (无严格顺序，仅表示关联)
  - **AI 推荐连线**: 可能有特殊动画或颜色提示

### 3.5 智能指令框 (Command Palette)

**功能**: 全键盘驱动的操作入口，集成 AI 智能伙伴。

**触发方式**:

- `Cmd/Ctrl + K`: 全局指令，在**工作空间**的任何位置触发。
- `/`: 在**任务卡片**编辑状态下快速输入。

**指令类型**:

- **创建**: "添加任务: 完成设计文档"
- **搜索**: "find: 设计文档" (搜索**任务卡片**、**知识节点**、**空间区域**)
- **AI 辅助**:
  - `/ai 帮我规划项目X的任务` (AI 智能生成并布局**任务卡片**和**空间区域**)
  - `/ai 将这些任务按优先级智能布局` (AI 对选定**任务卡片**进行自动排序和视觉重排)
  - `/ai 总结这个知识节点的内容`
- **导航**:
  - `go to: 项目A` (平移并聚焦到指定**空间区域**)
  - `go to: [任务卡片标题]` (平移并聚焦到指定**任务卡片**)
- **视图**:
  - `/view today` (激活“今日任务”**透视视图**，将今日任务智能布局到当前视口)
  - `/view kanban [空间区域名称]` (激活指定**空间区域**的看板**透视视图**)
  - `/view mindmap [中心节点]` (激活以某节点为中心的思维导图**透视视图**)
- **操作**: "完成任务"、"删除任务"、"更改状态: 进行中"
- **设置**: "设置主题为深色"

### 3.6 侧边栏 (Sidebar) - 快速导航与管理

**功能**: 提供快速访问和全局管理入口，而非多个独立的“项目画布”入口。

**结构**:

```
📁 快速访问与透视视图

├── ✨ AI 智能视图 (AI Generated Views)
│   ├── 今天的任务 (Today's Tasks) ← 基于条件过滤和 AI 布局的透视视图
│   └── 本周的计划 (This Week's Plan)
│
├── 🎯 收藏的透视视图 (Saved Perspective Views)
│   ├── 项目X看板
│   └── 我的学习路径图
│
├── 🏞️ 空间区域导航 (Space Navigation)
│   ├── 📁 产品迭代计划 (Project: Product Iteration)
│   │   ├── 📁 前端开发 (Space: Frontend Dev)
│   │   └── 📁 后端开发 (Space: Backend Dev)
│   ├── 📚 个人知识库 (Space: Personal Knowledge Base)
│   └── 💡 灵感收集 (Space: Ideas Collection)
│
├── 标签管理 (Tag Management)
│   ├── #重要
│   └── #紧急
│
└── ⚙️ 设置 (Settings)
    ├── 主题
    ├── 快捷键
    └── 数据同步
```

**UX 思考**:

- 侧边栏是用户快速切换不同**空间区域**（项目）或激活**透视视图**的入口。点击某个“空间区域”会平移并聚焦到该区域。点击一个“透视视图”会触发画布上相应的智能布局和筛选。
- “收藏的透视视图”允许用户保存常用的特定布局和筛选组合。

### 3.7 状态栏 (Status Bar)

**功能**: 提供系统状态信息和快速操作。

**内容**:

- 同步状态图标 (在线/离线，同步中/已同步)
- 当前**工作空间**中活跃**任务卡片**数量
- 通知和提示信息
- 快速切换**工作空间**主题 (未来多工作空间支持)

## 4. 交互设计

### 4.1 手势和快捷键 (Enhanced)

**工作空间操作**:

- `Space + 拖拽`: 平移画布。
- `滚轮`: 缩放。
- `Cmd/Ctrl + 0`: 重置缩放并居中所有可见内容。
- `Cmd/Ctrl + 1`: 缩放并适应所有**任务卡片**、**知识节点**和**空间区域**到屏幕。
- `Shift + 滚轮`: 左右滚动画布。
- `Cmd/Ctrl + F`: 打开搜索指令。
- `Alt/Option + 拖拽`: 复制选中的元素。

**任务卡片/知识节点操作**:

- `Enter`: 编辑**任务卡片/知识节点**。
- `Delete`: 删除选中元素。
- `Tab`: 在编辑状态下，快速完成当前编辑并创建下一个同级**任务卡片/知识节点**。
- `Shift + Enter`: 完成编辑并保存。
- `Cmd/Ctrl + D`: 切换**任务卡片**状态 (如完成/未完成)。

**选择操作**:

- `Cmd/Ctrl + A`: 全选所有可见元素。
- `Cmd/Ctrl + 点击`: 多选。
- `Shift + 点击`: 范围选择。

### 4.2 拖拽行为 (Enhanced)

**任务卡片/知识节点拖拽**:

- **单个拖拽**: 移动元素在**工作空间**中的位置。
- **多个拖拽**: 批量移动选中的元素。
- **拖拽到其他任务卡片/知识节点边缘**: 出现提示，自动创建**关系连线**（可选择连线类型）。
- **拖拽到空间区域内部**: 成为该**空间区域**的子元素，视觉上显示在框内。
- **拖拽到空间区域边缘**: 自动滚动画布，方便移动到远处区域。

**关系连线拖拽**:

- 从**任务卡片/知识节点**边缘拖出: 创建新的**关系连线**。
- 拖拽现有**关系连线**端点: 修改连接的目标元素。
- 双击**关系连线**: 编辑连线类型、添加描述等。

### 4.3 透视视图交互 (Crucial UX Handling)

- **激活视图**: 从侧边栏点击或通过指令框输入 `/view [类型/名称]`。
- **画布转换动画**: **工作空间**会平滑地缩放、平移，将符合条件的**任务卡片/知识节点**动画移动到屏幕中央的**聚焦区域**，并按该视图的规则（如看板、列表）进行布局。其他不相关的元素会平滑地淡出、缩小或移到背景层。
- **视图内操作**: 在**透视视图**中，**任务卡片/知识节点**仍可拖拽、编辑、更改状态。这些操作会更新其在当前视图布局中的临时位置，同时也会更新其在原始画布上的数据，并反映在所有相关的视图中。
- **退出视图**: 点击画布空白处、关闭侧边栏中的视图入口或输入 `/view all`。
- **画布恢复动画**: **任务卡片/知识节点**会平滑地动画返回到它们在**工作空间**上的原始位置和布局。这种“魔法般的”动画是降低心智负担的关键，让用户知道信息从未丢失，只是暂时换了个“视角”。

## 5. 响应式设计

### 5.1 桌面端 (>1200px)

- 完整的三栏布局 (侧边栏、工作空间、右侧详情/属性面板 - 可选)。
- 侧边栏常驻显示或可切换显示/隐藏。
- 支持多窗口操作 (例如，一个窗口聚焦某个项目空间区域，另一个窗口显示全局透视视图)。

### 5.2 平板端 (768px - 1200px)

- 可折叠侧边栏，通过滑动手势或点击按钮展开/收起。
- 触摸优化的控件尺寸和更大的点击目标。
- 支持多点触控手势进行画布操作（缩放、平移）。
- 任务卡片/知识节点可能采用更紧凑的显示模式。

### 5.3 移动端 (<768px)

- 底部导航栏，提供核心功能入口（如指令框、侧边栏开关、我的今日任务）。
- 侧边栏默认隐藏，从左侧滑出。
- **工作空间**聚焦在当前激活的**空间区域**或**透视视图**上。
- **任务卡片/知识节点**可能以更精简的列表或卡片形式显示，点击后展开详情。
- 指令框是主要的任务创建和视图切换方式。

## 6. 可访问性设计

### 6.1 键盘导航

- 所有功能都可通过键盘访问，包括画布平移、缩放、元素选择、编辑和视图切换。
- 清晰的焦点指示器 (高亮边框)。
- 逻辑的 Tab 顺序，符合用户预期。

### 6.2 屏幕阅读器支持

- 语义化的 HTML 结构，确保**任务卡片**、**空间区域**、**关系连线**等都能被屏幕阅读器正确识别和描述。
- 适当的 ARIA 标签，为复杂交互提供辅助信息。
- 描述性的替代文本，例如为图标和图片提供文字说明。

### 6.3 视觉辅助

- 高对比度模式。
- 可调节的字体大小。
- 色盲友好的配色方案，尤其是在**任务卡片**状态指示和**关系连线**类型上。

## 7. 性能考虑

### 7.1 渲染优化 (针对无限画布)

- **Canvas 分层渲染**: 将背景网格、空间区域、任务卡片、连线分别渲染在不同 Canvas 层，减少重绘区域。
- **虚拟化与视锥体裁剪**: 只渲染当前视口内可见的**任务卡片**、**知识节点**和**空间区域**。视口外的元素不渲染，仅存储数据。
- **增量渲染**: 只更新发生变化的区域，避免全画布重绘。
- **Web Worker**: 将复杂的空间索引计算、AI 布局算法放到 Web Worker 中，不阻塞主线程。

### 7.2 交互优化

- **防抖**: 搜索输入、快速连续操作等。
- **节流**: 滚动、缩放、拖拽等频繁事件。
- **预加载**: 预加载关键资源和常用数据。

## 8. 动效设计 (Emphasizing Smooth Transitions)

### 8.1 微交互

- **按钮/元素悬停**: 轻微放大 (scale: 1.05) 或背景色变化。
- **任务卡片创建**: 从指令框平滑弹出到画布，伴随从小到大的弹性动画。
- **状态切换**: 颜色渐变过渡，如完成任务时蓝色到灰色的平滑过渡。
- **连接点提示**: 拖拽创建连线时，目标卡片边缘出现连接点指示。

### 8.2 视图转场 (Crucial for UX)

- **画布平移/缩放**: 使用缓和曲线 (ease-out) 的平滑动画，避免生硬跳跃。
- **透视视图激活**:
  - 不相关元素: 渐进淡出、缩小或平滑移出当前视口。
  - 相关元素: 平滑地动画移动到**聚焦区域**，并按视图规则进行布局。
  - 整个过程应在 300-500ms 内完成，确保流畅的视觉连贯性。
- **退出透视视图**: 反向动画，任务卡片平滑地回到它们在**工作空间**中的原始坐标。

### 8.3 反馈动画

- **成功操作**: 绿色的勾选动画或短暂的底部通知。
- **错误提示**: 红色的摇摆动画或弹出的错误提示框。
- **加载状态**: 脉冲式的骨架屏或进度条。
- **AI 思考中**: 优雅的加载动画或状态文字。

## 10. 国际化设计 (i18n)

### 10.1 支持语言

**第一阶段**:

- 🇨🇳 简体中文 (zh-CN) - 默认语言
- 🇺🇸 英语 (en-US)
- 🇯🇵 日语 (ja-JP)

**第二阶段**:

- 🇰🇷 韩语 (ko-KR)
- 🇩🇪 德语 (de-DE)
- 🇫🇷 法语 (fr-FR)
- 🇪🇸 西班牙语 (es-ES)

### 10.2 文本处理策略

**静态文本**:

- 界面标签和按钮文字
- 菜单项和导航文本 (包括新的核心概念：**工作空间, 空间区域, 任务卡片, 知识节点, 关系连线, 透视视图**)
- 错误提示和系统消息
- 帮助文档和说明文字

**动态文本**:

- 用户创建的**任务卡片**内容、**知识节点**内容。
- **空间区域**名称和描述。
- 标签和分类名称。
- AI 生成的建议文本和回复。

### 10.3 布局适配

**文本长度处理**:

- 德语文本通常比英语长 30-50%。
- 中文/日语文本相对紧凑。
- 界面元素（按钮、输入框、卡片）需要支持动态宽度和高度，以适应不同语言文本长度变化。

**字体选择**:

```css
/* 多语言字体栈，确保涵盖常用 CJK 和拉丁字符 */
font-family:
  /* 中文 */ "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
  /* 日文 */ "Hiragino Kaku Gothic ProN", "Meiryo",
  /* 韩文 */ "Malgun Gothic", "Apple SD Gothic Neo",
  /* 西文 */ -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
  /* 后备 */ sans-serif;
```

**界面元素适配**:

- 按钮和输入框支持动态宽度。
- 菜单项高度自适应文本。
- 工具提示位置智能调整。

### 10.4 数据格式本地化

**日期时间**: 根据不同区域设置显示格式。
**数字格式**: 根据不同区域设置小数点和千位分隔符。
**相对时间**: 如“3分钟前”、“昨天”等本地化表达。

### 10.5 文化适配

- **颜色语义**: 谨慎使用具有强烈文化含义的颜色。例如，红色在中西方文化中的不同含义。
- **图标选择**: 优先使用国际通用的符号，避免使用具有文化特定含义的图标，并提供图标的文字说明。
- **交互习惯**: 考虑不同文化背景用户对信息密度、留白空间和交互流程的偏好。
